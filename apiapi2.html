Task Module API 명세서

기본 경로: /tasks

인증 필수: 모든 API는 JWT (Authorization: Bearer Token)를 요구합니다.

1. 할 일 CRUD 기능

No. 설명 경로 Method 필수 권한
T-01 할 일 등록 (Task Creation) /tasks/create POST 그룹장 (Leader)
T-02 할 일 상태 변경 /tasks/:taskId/status PATCH 담당자 또는 그룹장
T-04 할 일 수정 /tasks/:taskId PATCH 그룹장 (Leader)
T-05 할 일 삭제 /tasks/:taskId DELETE 그룹장 (Leader)

2. 할 일 조회 (T-03)

No. 설명 경로 Method 필수 권한
T-03 특정 방의 할 일 목록 조회 /tasks/room/:roomId GET 방 멤버 모두

3. 요청/응답 스키마 요약

T-01: 할 일 등록 (POST /tasks/create)

요청 Body (Input DTO):

{
    "roomId": 1, 
    "assignedUserId": 2,  // 할당된 멤버 ID (null 허용)
    "title": "주간 거실 청소",
    "description": "창틀 및 베란다 포함",
    "repeatRule": "WEEKLY" | "DAILY" | "NONE",
    "dueDate": "2025-10-28" // YYYY-MM-DD 형식
}

T-02: 할 일 상태 변경 (PATCH /tasks/:taskId/status)

요청 Body (Input DTO):

{
    "status": "DONE" | "TODO" | "IN_PROGRESS"
}

주요 로직: Task ID는 경로 파라미터(:taskId)로 전달하며, RoomRoleGuard와 TasksService에서 **Task 담당자** 또는 **그룹장**인지 확인합니다.

T-03: 할 일 목록 조회 응답 예시

응답 Body (Output DTO):

[
    {
        "id": 1,
        "title": "주간 거실 청소",
        "status": "TODO",
        "dueDate": "2025-10-28",
        "assignedUser": {
            "id": 2,
            "nickname": "kimkim" // 관계(relations)를 통해 담당자 닉네임 제공
        }
    }
]
