RoomLink API 인터페이스 명세서 (v1.0)

프로젝트 목표: 뉴스 이벤트 기반 변동성 예측을 통한 리스크 관리형 포트폴리오 코파일럿 개발 (FinSight 기반)

버전: 1.0 (Auth, Users, Room 핵심 기능 구현 완료)

인증 필요: Authorization: Bearer [JWT Token]

1. Users & Auth Module

No. 경로 Method 설명 권한
U-01 /users/signup POST 회원가입 (UsersService.createUser) 없음
U-02 /auth/login POST 로그인 및 JWT 토큰 발급 (AuthService.login) 없음
U-03 /users/me GET 내 프로필 조회 JWT 필수

U-02 로그인 요청 및 응답 예시

요청 Body:

{
    "email": "test@finsight.com",
    "password": "securepassword"
}

성공 응답 (200 OK):

{
    "access_token": "eyJhbGciOiJIUzI1NiI...",
    "user": {
        "id": 1,
        "email": "test@finsight.com",
        "nickname": "TestUser"
    }
}

2. Room Module (RoomLink 핵심 기능)

No. 경로 Method 설명 권한
1 /rooms/create POST 새로운 방을 생성하고, 요청 사용자를 **그룹장(leader)**으로 자동 등록 JWT 필수
2 /rooms/join POST 초대 코드(code)를 사용하여 방에 참여하고, **그룹원(member)**으로 자동 등록 JWT 필수
3 /rooms/me GET (구현 예정) 사용자가 속한 모든 방 목록과 해당 방에서의 역할(role) 조회 JWT 필수

R-01 방 생성 요청 및 응답 예시

요청 Body:

{
    "name": "우리 집 청소 그룹"
}

성공 응답 (201 Created):

{
    "message": "방 생성 및 그룹장 등록 성공",
    "room": {
        "id": 1,
        "name": "우리 집 청소 그룹",
        "code": "ABCDEFGH",
        "ownerId": 1
    }
}

R-02 방 참여 요청 및 응답 예시

요청 Body:

{
    "code": "ABCDEFGH"
}

성공 응답 (200 OK):

{
    "message": "방(우리 집 청소 그룹) 참여 성공",
    "room": {
        "id": 1,
        "name": "우리 집 청소 그룹",
        "code": "ABCDEFGH"
    }
}

3. 다음 구현 예정 모듈: Task Module

할 일 관리의 핵심 모듈로, P4 (LLM 피처 추출)의 기반이 될 데이터 구조를 포함합니다. 그룹장 권한(leader) 검증을 위한 RoomRoleGuard가 적용될 예정입니다.

주요 API Method 권한 설명
/tasks/create POST 그룹장 필수 새로운 할 일 등록 (반복 규칙, 담당자 지정 포함)
/tasks/:id/complete POST 담당자 필수 할 일 완료 처리 및 피드백/점수 반영 로직
